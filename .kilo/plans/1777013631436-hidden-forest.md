# Plan: Add DESIGN.md Reference as Static Resource

## Goal

Register the provided Google DESIGN.md announcement as a new static MCP resource so AI clients can access it via the server.

## Context

The MCP server currently serves 8 static workflow resources from `workflows/` via `src/resources/static.ts`, which iterates over `STATIC_RESOURCES` in `src/core/catalog.ts`. Adding a new static resource requires:

1. A markdown file in `workflows/`
2. An entry in `DOMAIN_FILES` (so readiness checks verify the file exists)
3. An entry in `STATIC_RESOURCES`
4. A type update in `ResourceDef.domain`
5. Test count updates

## Changes

### 1. Create `workflows/DESIGN_MD_REFERENCE.md`

Content: the user's provided DESIGN.md text (formatted as markdown).

### 2. Update `src/schemas/types.ts`

- Expand `ResourceDef.domain` from `Domain | 'execute-referencing'` to `Domain | 'execute-referencing' | 'design-reference'`.

### 3. Update `src/core/catalog.ts`

- Add `'design-reference': 'DESIGN_MD_REFERENCE.md'` to `DOMAIN_FILES`.
- Append a new `ResourceDef` to `STATIC_RESOURCES`:
  ```ts
  {
    id: 'design-md-reference',
    domain: 'design-reference',
    uri: 'workflow://design-reference/v4',
    fileName: 'DESIGN_MD_REFERENCE.md',
    title: 'DESIGN.md Reference',
    description: 'Google DESIGN.md open-source specification reference.',
    mimeType: 'text/markdown',
  }
  ```

### 4. Update `tests/unit/resources/registration.test.ts`

- Change static resource registration count from `8` → `9`.
- Change total resource registration count from `9` → `10`.

### 5. Update `internal-docs/modules.md`

- Update `STATIC_RESOURCES` description from "all 8 static resources" → "all 9 static resources".
- Update `src/resources/static.ts` description from "Registers 8 static resources" → "Registers 9 static resources".

## Validation

- Run `pnpm run test` to ensure registration tests pass.
- Run `pnpm run typecheck` to confirm type safety.
- Verify `getWorkflowReadiness()` includes the new file in its checks.
