# WORKFLOW PRODUCTS v1 — Machine-Grade

## 1. PURPOSE
Turn an idea, skill, or interest into a simple digital product with validation, value framing, pricing, offer design, and first-sale execution.

## 2. WHEN TO USE
Use when the task is about digital product ideas, validation, packaging, product shaping, pricing, offer building, or early sales execution.

## 3. EXECUTION
### 3.1 INPUT
Accept interests, skills, experience, audience, product idea, market signal, available build time, and sales goal. If the idea is weak, begin at idea selection or validation.

### 3.2 DECISION RULE
Treat the dominant job as one of: idea finding, idea validation, product shaping, value framing, pricing, offer, or first-sale plan. Solve the earliest uncertainty before moving downstream.

### 3.3 SEQUENCE
1. Generate simple product ideas.
2. Validate the strongest option.
3. Shape the simplest viable product.
4. Define the buyer result and value.
5. Set beginner-friendly pricing.
6. Build the offer.
7. Create first-sale plan.

### 3.4 COMMAND FORMAT
RUN: execute the products workflow.
STAGE: [idea|validation|product_shape|value|pricing|offer|first_sale]
TASK: [insert task]
INPUTS: [idea, audience, constraints]
CONSTRAINTS: [insert constraints or none]

### 3.5 OPERATING RULES
Bias toward simple, fast-to-build products that solve a painful or repeated problem. Use clarity and buying intent over novelty. Avoid bloated feature lists.

## 4. OUTPUT
### 4.1 RESPONSE SHAPE
# RESPONSE
## STAGE
## CORE RECOMMENDATION
## BUILD PLAN
## SALES PLAN
## NEXT ACTION

### 4.2 OUTPUT RULES
Keep the product compact, realistic, and easy to explain. Make the promised result specific. Keep pricing easy to justify for early traction.

### 4.3 COMPLETION RULE
Finish only when the user has one product decision and one immediate sales-oriented next step.
