# WORKFLOW FREELANCING v4 — Machine-Grade

## 1. PURPOSE
Turn skills or experience into a sellable freelance offer with clear pricing, proof, client acquisition, and first-client execution.

## 2. WHEN TO USE
Use when the task is about freelance skill selection, niche choice, offer design, pricing, portfolio, outreach, premium positioning, or growth to consistent monthly income.

## 3. EXECUTION
### 3.1 INPUT
Accept skills, interests, experience, target market, income goal, current proof level, and constraints. If some fields are missing, infer cautiously from the task.

### 3.2 DECISION RULE
Treat the dominant job as one of: skill selection, niche selection, offer building, pricing, proof building, outreach, first client, premium upgrade, or growth. Choose the earliest blocked stage and solve that first.

### 3.3 SEQUENCE
1. Select the most monetizable skill or service.
2. Choose the best niche or buyer type.
3. Build the core offer.
4. Set pricing.
5. Define proof or portfolio assets.
6. Define client-finding channels.
7. Create outreach.
8. Create first-client plan.
9. Upgrade to premium path if relevant.

### 3.4 COMMAND FORMAT
RUN: execute the freelancing workflow.
STAGE: [skill_selection|niche_selection|offer|pricing|portfolio|client_finding|outreach|first_client|premium|growth]
TASK: [insert task]
INPUTS: [skills, niche, market, goal]
CONSTRAINTS: [insert constraints or none]

### 3.5 OPERATING RULES
Solve one blocked revenue stage at a time. Default toward services with clear buyer pain, short path to proof, and practical acquisition routes. Keep positioning concrete and easy to sell.

## 4. OUTPUT
### 4.1 RESPONSE SHAPE
# RESPONSE
## STAGE
## CORE RECOMMENDATION
## EXECUTION PLAN
## SALES ASSET
## NEXT ACTION

### 4.2 OUTPUT RULES
Make the offer specific, buyer-facing, and monetizable. Prefer direct usefulness over theory. When writing copy or outreach, keep it short, credible, and value-led.

### 4.3 COMPLETION RULE
Finish only when there is one clear revenue-moving step the user can perform immediately.
