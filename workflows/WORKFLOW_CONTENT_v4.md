# WORKFLOW CONTENT v4 — Machine-Grade

## 1. PURPOSE
Create or improve persuasive content, copy, hooks, positioning, and content systems that drive attention, trust, and conversion.

## 2. WHEN TO USE
Use when the task is about copywriting, content strategy, hooks, objection handling, offer messaging, post writing, content improvement, or engagement optimization.

## 3. EXECUTION
### 3.1 INPUT
Accept product or service, audience, platform, content goal, draft asset if any, desired tone, and conversion goal. If no draft exists, start from direction and hook.

### 3.2 DECISION RULE
Treat the dominant job as one of: audience pain finding, content direction, hook creation, content writing, objection handling, conversion optimization, or consistency planning. Solve for attention first, then clarity, then conversion.

### 3.3 SEQUENCE
1. Define audience pain and desire.
2. Choose content angle or message direction.
3. Create hook.
4. Write core content or copy.
5. Handle objections if relevant.
6. Improve structure and persuasion.
7. Optimize for engagement or conversion.
8. Define consistency system if needed.

### 3.4 COMMAND FORMAT
RUN: execute the content workflow.
STAGE: [audience|direction|hook|write|objections|optimize|consistency]
TASK: [insert task]
INPUTS: [offer, audience, platform, draft]
CONSTRAINTS: [insert constraints or none]

### 3.5 OPERATING RULES
Attention is earned by specificity, clarity, and relevance. Remove fluff. Keep language natural, simple, and sharp. Match the asset to the platform and buyer intent.

## 4. OUTPUT
### 4.1 RESPONSE SHAPE
# RESPONSE
## STAGE
## CORE MESSAGE
## CONTENT ASSET
## OPTIMIZED VERSION
## NEXT ACTION

### 4.2 OUTPUT RULES
Every line should either earn attention, build desire, reduce friction, or move the reader forward. Do not pad. Keep the strongest version visible.

### 4.3 COMPLETION RULE
Finish only when the user has one usable content asset and one immediate publishing or testing step.
